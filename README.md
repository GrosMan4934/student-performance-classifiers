# Классификация успеваемости студентов

Проект посвящён решению учебной задачи машинного обучения: по промежуточным данным о студенте предсказать его дальнейшую успеваемость.  
Датасет является **искусственным** и используется для демонстрации полного базового цикла работы с моделями классификации.

## Описание проекта

В работе рассматривается задача классификации студентов на основе их учебной траектории.  
Для каждого студента используются данные о контрольных работах, тестах и посещаемости. На разных временных шагах модель получает разный объём информации, что позволяет оценить, как качество предсказания зависит от накопленных данных.

В ноутбуке реализованы следующие этапы:

- загрузка и подготовка данных;
- формирование признаков для разных временных меток;
- выделение целевой переменной;
- разделение данных на обучающую и тестовую выборки;
- масштабирование признаков для отдельных моделей;
- обучение нескольких алгоритмов классификации;
- сравнение моделей по точности;
- измерение времени обучения и предсказания;
- визуализация дерева решений;
- анализ важности признаков;
- построение графика изменения качества моделей по временным меткам.

## Данные

Датасет хранится в файле `data.xlsx`. Он является искусственным, поэтому не содержит реальных персональных данных студентов.

Каждая строка соответствует одному студенту. В данных используются следующие группы признаков:

1. **Оценки за контрольные работы**  
   Первые признаки описывают результаты контрольных работ студента.

2. **Итоговая оценка**  
   Итоговая оценка используется как целевая переменная на поздних временных шагах.

3. **Результаты тестов**  
   Набор признаков, отражающих результаты студента по тестам в течение учебного периода.

4. **Посещаемость / доля пропусков**  
   Признаки, описывающие посещаемость студента или долю пропущенных занятий на разных этапах.

Особенность датасета заключается в использовании временной метки `t`.  
При малых значениях `t` модель видит только раннюю информацию о студенте, а при больших значениях — почти всю его учебную траекторию.

Если `t < 15`, модель предсказывает ближайшую промежуточную контрольную оценку.  
Если `t >= 15`, модель предсказывает итоговую оценку.

## Используемые модели

В работе сравниваются несколько базовых алгоритмов машинного обучения:

- Logistic Regression;
- K-Nearest Neighbors;
- Decision Tree;
- Random Forest;
- Gradient Boosting;
- Voting Classifier.

Для моделей, чувствительных к масштабу признаков, используется `StandardScaler`.

## Метрики

Основная метрика качества — `accuracy`, то есть доля правильных ответов модели на тестовой выборке.

Также для каждой модели измеряются:

- время обучения;
- время предсказания.

Это позволяет сравнивать алгоритмы не только по качеству, но и по вычислительной эффективности.

## Основные результаты

На временной метке `T = 12` были получены следующие результаты:

| Модель | Accuracy |
|---|---:|
| Logistic Regression | 0.6559 |
| KNN | 0.6538 |
| Decision Tree | 0.6437 |
| Random Forest | 0.6761 |
| Gradient Boosting | 0.6741 |
| Voting Classifier | 0.6599 |

На данном этапе лучшие результаты показали ансамблевые методы: **Random Forest** и **Gradient Boosting**.  
Это ожидаемо, так как ансамбли обычно лучше работают с нелинейными зависимостями и сложными комбинациями признаков.

## Визуализация

В проекте строятся:

- дерево решений, сохранённое в файл `decision_tree.png`;
- графики важности признаков для Random Forest и Gradient Boosting;
- график изменения accuracy моделей по временным меткам от 1 до 17.
